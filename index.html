JIRA를 통한 업무관리

저는 프로젝트 협업 관리 툴 JIRA 로 퍼블리싱 업무관리를 하고 있습니다.

※JIRA 란 ?
JIRA는 Atlassian사에서 개발한 Issue Tracking System 입니다.
말 그대로 이 "이슈 추적 시스템"은 원래 버그 추적 을 위해 만들어졌지만, 많이 발전되어 현재는 버그 뿐만 아니라 일반적인 이슈에서부터 프로젝트 스케쥴에 대한 관리와 개발자간에 작업을 배분하고 커뮤니케이션하는 도구로 사용됩니다.
한마디로 '개발 협업 관리 툴'

저는 다음과 같은 이슈생성 규칙을 정의하고 실무에 적용하고 있습니다.
키
퍼블리싱(마크업) 업무 키 :
DSMU (Donga Science MarkUp) 로 지정하여 관리
제목
업무명 생성 규칙 :
[DSMU-번호][서비스명][디바이스 환경] 간략한 업무 제목
ex) [DSMU-23] [kids] [pc] 우측 지사탐 카운트다운 배너 추가 작업
내용
이슈, 작업 진행 내용, 특이사항 등 정리
고정 필드
진행 상태(To do / In progress / finish), 담당자(실 작업 진행자), 보고자(이슈 발행자), start date, end date

*현재는 퍼블리셔가 혼자이기 때문에 JIRA와 같은 프로젝트 관리 툴이 필요하진 않지만 회사가 성장하여 개발 팀 규모가 커졌을 때를 대비해 기반을 만들고 사전 연습중입니다.

서비스개발팀

동아사이언스
